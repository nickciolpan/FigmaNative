How It Works
Three steps to production code

No boilerplate. No copy-paste. Just select in Figma and press export.

Configure once

Drop a config file in your project root. Tell FigmaNative your style system and theme tokens.

// figmanative.config.js
mode: "stylesheet"
theme.hook: "useColors"

Start the bridge

One command starts the local server that connects the Figma plugin to your codebase.

npx figma2native serve

Export from Figma

Select a component or frame in Figma, click "Export to React Native". Code appears in your project.

src/components/Card.tsx ✓

Sync back

Click "Sync from Code" in the plugin. Your React Native components appear as Figma components with full fidelity.

Figma ← 6 components synced
Style Backends
Your style system, your rules

Same Figma design, different output. Choose the backend that matches your project.

NativeWind Tailwind

Generates className strings with Tailwind utility classes. Perfect for NativeWind projects.

import { View, Text } from "react-native"; export function Card() { return ( <View className="p-4 gap-3 rounded-lg bg-gray-50 items-center"> <Text className="text-lg font-bold text-gray-900"> Title </Text> </View> ); }

StyleSheet Theme-Aware

Generates StyleSheet.create() with useMemo and your theme hook. Matches existing codebases.

const Colors = useColors(); const styles = React.useMemo(() => StyleSheet.create({ container: { padding: 16, gap: 12, borderRadius: 8, backgroundColor: Colors.card, }, }), [Colors]);
