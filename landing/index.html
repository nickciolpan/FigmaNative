Features

Everything you need to ship

From design to production-ready code with intelligent defaults and full customization.

◈

AI Code Generation

Claude analyzes your Figma layers, styles, and structure to produce clean, idiomatic React Native components with proper typing.

⚙

Asset Pipeline

Images and icons are automatically exported from Figma, optimized, and placed in your project with correct import paths.

☘

Theme-Aware Colors

Maps Figma colors to your theme tokens. Supports light/dark mode with configurable hook and import paths.

⚙

Live Preview

See generated components rendered instantly in Expo Go or your development build with hot reload support.

✎

Designer Prompts

Designers can add instructions directly in the Figma plugin. Notes like "use FlatList" are sent to the AI as context.

✓

Code Validation

Generated code is validated for syntax errors, missing imports, and style consistency before being written to disk.

How It Works

Four steps to production code

Set up once, then export any component from Figma with a single click.

1

Configure

Create a figmanative.config.js in your project root with theme colors, output paths, and AI settings.

2

Start Bridge

Run the local bridge server that connects Figma to your project and Claude.

3

Install Plugin

Load the FigmaNative plugin in Figma from the community or install locally for development.

4

Select & Export

Select any frame or component in Figma, add optional notes, and click Export. Your TSX file appears instantly.

Configuration

Fully customizable

One config file controls themes, output, AI behavior, prompts, and more.

figmanative.config.js 
module.exports = {
  theme: {
    colors: {
      primary: '#6366F1',
      background: '#FFFFFF',
      text: '#09090B',
      border: '#E4E4E7',
    },
    darkColors: {
      primary: '#818CF8',
      background: '#09090B',
      text: '#FAFAFA',
      border: '#27272A',
    },
    hook: 'useTheme',
    import: '@/theme',
  },

  components: {
    dir: 'src/components',
    index: true,
  },

  ai: {
    claudePath: 'claude',
    model: 'sonnet',
    timeout: 60000,
  },

  prompts: {
    styling: 'Use StyleSheet.create for all styles',
    structure: 'One component per file, named export',
    quality: 'Add proper TypeScript types for all props',
    output: 'Return only the component code, no markdown',
    extra: 'Support both light and dark themes',
  },

  server: {
    port: 3001,
  },
};
Get Started

Up and running in minutes

Seven steps from zero to generating your first component.

1

Clone the repository

$ git clone https://github.com/nickciolpan/FigmaNative.git
2

Install CLI dependencies

$ cd FigmaNative/cli && npm install
3

Create your config file

Add a figmanative.config.js in your React Native project root with your theme and paths.

4

Start the bridge server

$ node cli/bridge.js --config ./figmanative.config.js
5

Install the Figma plugin

In Figma, go to Plugins → Development → Import plugin from manifest and select figma-plugin/manifest.json.

6

Select a component in Figma

Click on any frame or component you want to export. Add optional designer notes in the plugin panel.

7

Click Export

The plugin sends the design to Claude via the bridge. Your generated .tsx file appears in your components directory.
